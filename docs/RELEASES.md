## 一、版本号规范

采用 [SemVer](https://semver.org/) 标准：

```
主版本号.次版本号.修订号-预发布标识+构建元数据
   │       │       │         │           │
   │       │       │         │           └─ 构建号 (可选)
   │       │       │         └─ beta/rc (可选)
   │       │       └─ 向后兼容的问题修复
   │       └─ 向后兼容的功能新增
   └─ 不兼容的API修改
```

**示例:**
- `0.1.0` — 初始版本
- `0.2.0` — 新增语料库功能
- `0.2.1` — 修复上传白屏bug
- `1.0.0-beta.1` — 1.0首个测试版
- `1.0.0` — 正式版

## 二、发布流程

```
确定版本号 → 更新CHANGELOG → 打Git标签 → 部署 → 通知
      │             │              │          │        │
   评估变更      记录所有       标记       Vercel/   飞书群
   类型(scope)   修改点         发布点     Render    发布说明
```

### 2.1 版本号确定规则

| 变更类型 | 版本号变化 | 示例 |
|----------|------------|------|
| Bug修复 | 修订号+1 | `0.2.0` → `0.2.1` |
| 新功能（向后兼容） | 次版本号+1 | `0.2.1` → `0.3.0` |
| 破坏性变更 | 主版本号+1 | `0.3.0` → `1.0.0` |
| 紧急热修复 | 修订号+1，加`-hotfix` | `0.2.1-hotfix.1` |

### 2.2 Git标签规范

```bash
# 发布时打标签
git tag -a v0.2.1 -m "Release v0.2.1: 修复语料库上传白屏问题"

# 推送标签到远程
git push origin v0.2.1

# 查看所有标签
git tag -l
```

**标签格式:**
- 轻量标签: 不使用，必须用带注释的标签 (`-a`)
- 标签名: `v` + 版本号 (如 `v0.2.1`)
- 标签信息: 必须包含版本号 + 一句话摘要

### 3.3 变更分类

| 分类 | 说明 | 用户可见 |
|------|------|----------|
| `Added` | 新增功能 | ✅ |
| `Changed` | 功能变更 | ✅ |
| `Deprecated` | 即将废弃 | ✅ |
| `Removed` | 移除功能 | ✅ |
| `Fixed` | Bug修复 | ✅ |
| `Security` | 安全修复 | ✅ |
| `Tech` | 技术债务/重构 | ❌ |
